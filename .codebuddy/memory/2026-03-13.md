# 2026-03-13 工作日志

## 修复：消息处理崩溃导致"处理消息时出错了，请稍后重试"

### 问题现象
- 用户给 Karvis 发消息时收到"处理消息时出错了，请稍后重试"
- 消息内容包含 `@image:image_1773366137202.png` 格式文本

### 根因分析
- "处理消息时出错了"来自 `app.py` 第 752 行 `handle_message()` 的最外层异常捕获
- `brain.process()` 函数没有 try-except 保护，内部任何异常直接上抛
- LLM API 响应解析使用了不安全的 `result["choices"][0]["message"]["content"]` 索引方式
  - 当 Gemini/DeepSeek/Qwen 返回空 choices（如安全过滤）时会 IndexError
  - `call_llm` 虽然有 try-except，但某些边缘情况仍可能泄漏异常

### 修改内容

#### brain.py
1. **LLM 响应解析安全化**（3处）
   - `_call_deepseek()`：改用 `.get()` 安全访问 choices/message/content
   - `_call_qwen_flash()`：同上
   - `_call_gemini()`：同上
   - 空 choices 或空 content 时返回 None 并记录详细日志

2. **brain.process 异常保护**
   - 将原 process() 核心逻辑提取为 `_process_inner()`
   - process() 外层包裹 try-except，异常时：
     - 记录详细日志（异常类型 + traceback）
     - 尝试兜底保存消息到 Quick-Notes
     - 返回友好回复而非触发上层"处理消息时出错了"

#### app.py
3. **增强异常日志**
   - 在 handle_message except 中增加异常类型名 `type(e).__name__`
   - 增加 msg_type 和 msg_keys 信息，方便定位是哪种消息触发的问题
